🧪 Teste Chat em Tempo Real

Guia passo a passo para testar se o chat está funcionando corretamente.

📋 Checklist Rápido (5 minutos)

Backend rodando na porta 3000
Frontend rodando na porta 3001
Consegue fazer login
Browser aberto em localhost:3001
DevTools aberto (F12)

🚀 Passos do Teste

✅ Passo 1: Verificar Socket.io

Abra o DevTools (F12) e vá na aba Console.

Você deve ver uma mensagem como:

✅ Socket.io conectado

Se NÃO aparecer, o Socket.io não conectou. Verifique se o backend está rodando.

✅ Passo 2: Abrir o Chat

Clique no botão 💬 no canto inferior direito da tela.

No console, você deve ver:

✅ Entrado na sala: chat:abc123...

Se aparecer um erro no console, veja a seção de Troubleshooting.

✅ Passo 3: Digitar uma Mensagem

Na caixa de texto do chat, escreva algo como "Oi, tudo bem?"

Observe:

  • O botão ✓ fica azul (ativo)
  • A caixa de texto aceita seu texto
  • No console pode aparecer digitação (opcional)
✅ Passo 4: Enviar a Mensagem

Clique no botão ✓ ou aperte Enter.

Aguarde 1-2 segundos. Você deve ver no console:

✅ Mensagem enviada com sucesso: {_id: '...', text: 'Oi, tudo bem?', ...}

E na tela:

  • A caixa de texto fica vazia
  • Sua mensagem aparece no histórico acima
✅ Passo 5: Testar em Outro Navegador

Abra uma segunda aba do navegador:

  • Se você é Cliente → abra como Loja
  • Se você é Loja → abra como Cliente

Faça login com outra conta (se houver) e abra o chat.

Você deve ver a mensagem que enviou na aba 1 aparecer em tempo real na aba 2!

✅ Se viu a mensagem aparecer em tempo real
Parabéns! O chat funciona corretamente! 🎉
✅ Bônus: Testar Typing Indicator

Na segunda aba, comece a digitar algo.

Na primeira aba, você deve ver:

Loja está digitando... ⏳

Com 3 pontos pulsando: . . .

Quando parar de digitar, desaparece em 1 segundo.

🐛 Troubleshooting

❌ Erro: "Socket.io desconectado"

Backend não está rodando ou a porta 3000 está bloqueada.

Solução:

cd D:\PROJETOS\Drop && npm start
❌ Erro: "Não consegue abrir o chat"

A API de criar conversa está retornando erro.

Verifique no Network (F12 → Network):

  • POST /chat/conversations/pre-purchase
  • Status deve ser 200 (verde)
  • Se for 401, token expirou → faça login de novo
  • Se for 500, backend tem bug → ver logs
❌ Erro: "Chat abre mas não consegue enviar mensagem"

Este era o bug que corrigimos! ✅

Verifique se há atualização do navegador.

Faça: Ctrl+Shift+R (refresh hard)

⚠️ Aviso: Mensagem aparece na tela mas não no outro browser

Socket.io pode não estar emitindo o evento corretamente.

Solução:

  • Verificar Network → WS → Messages (procurar por chat:message)
  • Ver logs do backend (terminal onde npm start está rodando)
  • Reiniciar backend e frontend
